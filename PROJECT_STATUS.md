## 1. 项目概述

### 1.1 项目名称
AI World - AI任务编排与执行系统

### 1.2 项目类型
2D俯视视角模拟游戏 + AI工作流编排器

### 1.3 核心定位
基于AI规划师的任务编排与执行系统，采用游戏化界面展示AI任务处理流程。系统通过可视化方式呈现AI规划师分析需求、分解任务并调度AI员工执行的工作机制。

### 1.4 游戏场景
- **工作区**: 6个工位，员工在此执行任务
- **休息区**: 沙发、绿植，员工休息和等待区域
- **走廊**: 连接工作区和休息区的通道
- **老板**: 玩家控制的角色，可移动并与员工交互

## 2. 系统架构

### 2.1 AI员工团队（6个角色）

| ID | 姓名 | 默认模型 | 擅长领域 | API平台 |
|----|------|---------|----------|---------|
| 0 | 小明 | qwen-max | 通用复杂任务 | 阿里云百炼 |
| 1 | 小红 | qwen3.6-plus | 多模态处理 | 阿里云百炼 |
| 2 | 阿强 | qwen3-vl-235b-thinking | 视觉理解分析 | 阿里云百炼 |
| 3 | 小丽 | qwen-coder-turbo-0919 | 代码生成与审查 | 阿里云百炼 |
| 4 | 大伟 | qvq-max-2025-03-25 | 逻辑推理与决策 | 阿里云百炼 |
| 5 | 晓晓 | qwen-plus | 通用均衡任务 | 阿里云百炼 |

**员工状态**:
- 等待分配（初始状态）
- 工作中（在工位上）
- 休息中（在休息区）
- 行走中（移动中）
- 坐着（休息区坐着）

### 2.2 工作模式

**仅支持并行模式**:
- 所有选中的员工同时开始工作
- 各自独立完成任务
- 无任务依赖关系
- 各自完成后立即返回休息区

### 2.3 AI规划流程

```
用户输入需求
    ↓
AI规划师分析需求
    ↓
为每个员工生成角色分工
    ↓
生成技能文件(skill/skillX.md)
    ↓
用户按空格发布号令
    ↓
员工前往工位（或传送）
    ↓
并行执行AI任务
    ↓
实时显示进度
    ↓
任务完成，保存成果
    ↓
格式化为Markdown
    ↓
生成MERGED_RESULT.md
    ↓
员工返回休息区
```

## 3. 快速开始

### 3.1 环境要求
- Python 3.10+
- Windows 10/11

### 3.2 安装依赖
```bash
pip install -r requirements.txt
```

### 3.3 启动游戏
```bash
py main.py
```

### 3.4 创建项目

1. **打开流程编辑器** - 按 `P` 键
2. **选择员工** - 点击员工按钮（绿色边框为选中）
3. **输入需求** - 在输入框中输入提示词
4. **AI规划** - 点击"开始规划"
5. **发布号令** - 按 `空格` 键开始执行

### 3.5 查看成果

成果文件位置:
```
results/
└── [公司名称]_[项目名]_[时间戳]/
    ├── README.md              # 项目报告
    ├── MERGED_RESULT.md       # 整合成果
    ├── project_meta.json      # 项目元数据
    └── [编号]_[角色].md       # 各员工成果
```
